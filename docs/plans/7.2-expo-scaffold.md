# 7.2 Expo Scaffold — Implementationsplan

## Context

Monorepo-omstruktureringen (7.1) är klar. `packages/app/` är en tom placeholder. Nu ska vi scaffolda ett Expo-projekt som bevisar att delad data flödar från `@odlingsguiden/shared` till en native app. Minimalt men fungerande — inga fullständiga skärmar, bara bevis på att allt hänger ihop.

## Branch

`feature/expo-scaffold`

## Steg

### 1. Initiera Expo-projekt

```bash
cd packages/app
npx create-expo-app@latest . --template blank-typescript
```

Sedan rensa genererade filer som inte behövs och anpassa `package.json`:
- `"name": "@odlingsguiden/app"`
- `"main": "expo-router/entry"`
- `"@odlingsguiden/shared": "*"` som dependency

### 2. Installera dependencies

```bash
npx expo install expo-router expo-font expo-splash-screen \
  @react-native-async-storage/async-storage \
  react-native-safe-area-context react-native-screens react-native-svg
```

Sedan `npm install` från monorepo-root för workspace-symlinks.

### 3. Metro-config för workspaces

`packages/app/metro.config.js` (CommonJS — Metro kräver det):
- `watchFolders: [monorepoRoot]` — ser `packages/shared/src/`
- `nodeModulesPaths` — app + root node_modules
- `unstable_enableSymlinks: true` — följer workspace-symlinks
- Fallback: `resolveRequest` om symlinks krånglar

### 4. app.json

- `scheme: "odlingsguiden"` (krävs av Expo Router)
- `experiments.typedRoutes: true`
- `orientation: "portrait"`

### 5. tsconfig.json

Extend `expo/tsconfig.base` (inte monorepo-basen — Expo har egna RN-inställningar).
`paths`-mapping för `@odlingsguiden/shared` → `../shared/src/index.ts`.

### 6. Expo Router filstruktur

```
app/
  _layout.tsx           — Root: fonts (Fraunces + Lora), ZoneProvider, Stack
  (tabs)/
    _layout.tsx         — Tab bar: Grödor | Säsong
    index.tsx           — Grödlista (FlatList med CROP_LIST från shared)
    sasong.tsx          — Stub
  crop/
    [id].tsx            — Grödprofil-stub (getCachedCrop från shared)
  zone.tsx              — Zonväljare-stub (modal)
```

### 7. Theme-adapter

`packages/app/src/theme/tokens.ts` — tunt lager som konverterar shared:s CSS-strängar (`'4px'`) till RN-numbers (`4`). `colors` och `fontWeights` re-exporteras rakt av.

### 8. ZoneContext + AsyncStorage

`packages/app/src/hooks/useZone.tsx` — React Context med `AsyncStorage` istället för `localStorage`. Default zon 2. Wrappas i `_layout.tsx`.

### 9. Fonter

Kopiera `.ttf` från `@fontsource/fraunces` och `@fontsource/lora` (redan installerade i web) till `packages/app/assets/fonts/`. Ladda med `useFonts()` i root layout.

Varianter: Fraunces 400/700/800, Lora 400/700.

### 10. Root scripts

Lägg till i root `package.json`:
```json
"start:app": "npm run start -w packages/app"
```

## Verifiering

1. `npm install` från root utan fel
2. `cd packages/app && npx expo start`
3. Metro bundlar utan "could not find module" errors
4. Appen startar i Expo Go / simulator
5. Grödlistan visar 85+ grödor från `CROP_LIST`
6. Tap på gröda → detaljskärm med namn från `getCachedCrop`
7. Fonter (Fraunces/Lora) renderas korrekt

## Noteringar

- Node 18+ krävs (nvm tillgängligt)
- Expo SDK 53 med React 19 — matchar web-paketet
- Shared exporterar raw `.ts` — Metro hanterar det via Babel
- `metro.config.js` måste vara CJS trots `"type": "module"` i root
